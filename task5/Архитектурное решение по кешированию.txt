Мотивация
 Кэширование внедряем для:
  1. Увеличения скорости работы Front и Back приложений
  2. Снижения нагрузки на сеть
  3. Снижения нагрузки на сервисы и БД 
 Решаем проблемы:
  1. Убираем проблему низкой скорости (снижем время отклика) работы Front приложений
  2. Убираем проблему (не во всех) низкой скорости работы backend'a
  3. Повышаем отказоустойчивость (из-за меньшей нагрузки на hardware)
 В кэширование планирую включать:
    InternetShop, ShopAPI
    CRM API
    MES API 
Предлагаемое решение 
 1 Будем внедрять клиентское и серверное кэширование
 2 Клиентское кэширование для InternetShop (статический контент)
   Серверное кэширование для ShopAPI, CRM API, MES API - кэширование данных из БД для быстрого ответа
3 Если вы решите куда-то внедрить серверное кеширование, то поясните, какой паттерн будете применять — 
   ShopAPI
   CRM API
   MES API
      Cache-Aside (кэширование справочных данных, списка изделий, корзина и тп), высокая производительность на чтение, операций записе не много.
      Write-Through - не используем, низкая скорость
      Refresh-Ahead - сложная и требует мониторинга
5 Сиквенс диаграмма в файле cache_aside.puml
